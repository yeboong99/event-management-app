# UI/UX 개선 사항

> Playwright E2E 테스트(2026-03-22) 중 발견된 UI/UX 개선 필요 사항을 기록합니다.
> 심각도: `긴급` > `높음` > `중간` > `낮음`
> **이 문서는 개선 계획 목록이며, 즉시 수정하지 않습니다.**

## 긴급

### [UI-001] 로그인 후 주최자/참여자 역할 전환 진입점 부재

**심각도**: 긴급
**발생 위치**: 로그인 완료 시, 전체 앱

**현상**

- 로그인 후 참여자 뷰(`/discover`)로 리다이렉트됨
- 주최자 뷰(`/home`)로 이동하려면 URL을 직접 입력해야 함
- 앱 내 어느 화면에도 "주최자 모드"↔"참여자 모드" 전환 버튼이 없음

**문제점**

- 일반 사용자는 주최자 기능(이벤트 생성·수정·삭제)에 접근하기 어려움
- 두 역할을 오가는 사용자가 URL 구조를 알아야만 탐색 가능

**개선 방향 (채택)**

- `(host)` / `(participant)` 분리 라우트 그룹 구조를 **통합 `(app)` 라우트 그룹**으로 전환
- 역할 전환 버튼 대신 **통합 하단 탭 5개** (탐색 / 내 활동 / 만들기 / 카풀 / 프로필)로 역할 구분 자체를 제거
- `/my-events` 페이지 내 "참여 중 | 주최 중" 서브탭으로 참여자/주최자 기능 통합
- 단일 진입점: 로그인 후 `/discover`로 통일 (`/home` 폐기, `/discover`로 리다이렉트)
- **근거:** Meetup, Eventbrite 등 실제 이벤트 앱은 역할 전환 개념 없이 기능 탭으로 통합

## 높음

### [UI-002] 카테고리 탭 영역 스크롤바 및 테두리 라운딩 문제

**심각도**: 높음
**발생 위치**: `/events`, `/discover` 페이지 카테고리 탭 영역

**현상**

- 카테고리 탭 목록(전체·생일파티·파티모임 등) 아래 가로 스크롤바 표시
- 탭 우측에 세로 스크롤 화살표(▲▼) 표시
- 탭 컨테이너의 좌측 하단·우측 하단·우측 상단 라운딩(border-radius) 처리 불완전

**문제점**

- 의도하지 않은 스크롤바 노출로 UI가 지저분해 보임
- 모바일 앱처럼 보여야 하는 디자인 컨셉과 불일치

**개선 방향**

- 탭 컨테이너에 `overflow: hidden` 또는 `scrollbar-width: none` 적용
- 가로 스크롤은 터치 슬라이드(스와이프)로 구현하고 스크롤바 숨김
- 컨테이너 border-radius 일관성 확보
- 최신 트렌드: iOS/Android 네이티브 앱 스타일의 스크롤 없는 필터 칩(Chip) UI 적용

### [UI-003] 전체 페이지 우측 세로 스크롤바 노출

**심각도**: 높음
**발생 위치**: `/home`, `/events`, `/events/[eventId]`, `/events/[eventId]/edit` 등 전체 페이지

**현상**

- 콘텐츠가 뷰포트 높이 이하임에도 우측에 세로 스크롤바가 항상 표시됨
- 모바일 앱 레이아웃을 모방하는 디자인이지만 웹 브라우저 스크롤바가 노출됨

**문제점**

- 모바일 앱처럼 보여야 하는 디자인 컨셉 훼손
- 불필요한 스크롤바로 레이아웃 여백이 비대칭으로 보임

**개선 방향**

- 글로벌 CSS에서 `overflow: hidden` 또는 `::-webkit-scrollbar { display: none }` 적용
- 스크롤이 필요한 콘텐츠 영역에만 선택적으로 `overflow-y: auto` 적용
- 최신 트렌드: PWA 앱처럼 스크롤바를 숨기고 스와이프 제스처로 탐색

## 중간

### [UI-004] 404 페이지 영문 표시

**심각도**: 중간
**발생 위치**: 존재하지 않는 경로 접근 시

**현상**

- 잘못된 URL 접근 시 Next.js 기본 404 페이지("This page could not be found.") 영문 표시
- 앱 레이아웃(헤더·네비게이션)은 적용되지만 콘텐츠는 기본 영문 메시지

**문제점**

- 한국어 서비스에서 영문 에러 메시지는 사용자 경험 저하
- 뒤로 가거나 홈으로 이동하는 안내 버튼 없음

**개선 방향**

- `app/not-found.tsx` 커스텀 404 페이지 생성
- 한국어 메시지 + 홈 이동 버튼 제공
- 최신 트렌드: 친근한 일러스트레이션 + "홈으로 돌아가기" CTA 버튼

### [UI-005] 앱 탭 타이틀이 스타터킷 기본값으로 표시

**심각도**: 중간
**발생 위치**: 브라우저 탭 전체

**현상**

- 모든 페이지의 브라우저 탭 제목이 "Next.js and Supabase Starter Kit"으로 표시됨
- 페이지별 구분 없이 동일한 타이틀

**문제점**

- 서비스 브랜딩 미반영
- 여러 탭을 열었을 때 어떤 페이지인지 구분 불가

**개선 방향**

- `app/layout.tsx`의 metadata에서 `title` 업데이트 ("이벤트 매니저")
- 각 페이지에서 `generateMetadata`로 페이지별 타이틀 설정
  - 예: "Playwright 테스트 이벤트 | 이벤트 매니저"

### [UI-006] 이벤트 수정 시 일시가 UTC 기준으로 표시

**심각도**: 중간
**발생 위치**: `/events/[eventId]/edit` 수정 페이지

**현상**

- 이벤트 생성 시 KST(한국 표준시) 19:00으로 입력했으나
- 수정 페이지에서는 UTC 10:00으로 표시됨 (9시간 차이)

**문제점**

- 사용자가 시간을 다시 확인해야 하며, 잘못 수정할 위험
- 한국 사용자 대상 서비스에서 UTC 표시는 혼란 유발

**개선 방향**

- DB 저장 시 또는 읽기 시 KST 변환 처리
  - 클라이언트 사이드: `new Date(event_date).toLocaleString('ko-KR')` 활용
  - 또는 UTC+9 오프셋을 적용한 로컬 datetime 문자열 생성
- 일시 표시 전반을 KST 기준으로 통일

### [UI-007] 상세 페이지 뒤로가기 버튼 부재

**심각도**: 중간
**발생 위치**: `/events/[eventId]` 상세 페이지

**현상**

- 이벤트 상세 페이지에 뒤로가기(←) 버튼이 없음
- 목록으로 돌아가려면 하단 내비게이션의 "내이벤트" 탭을 눌러야 함

**문제점**

- 모바일 앱 UX 기준에서 상세 페이지에 뒤로가기 버튼은 필수 요소
- URL 직접 접근(딥링크) 사용자는 이전 맥락으로 돌아가기 어려움

**개선 방향**

- 커버 이미지 위 또는 헤더 좌측에 뒤로가기 버튼(`router.back()`) 추가
- 최신 트렌드: iOS/Android 스타일의 반투명 뒤로가기 버튼(커버 이미지 위에 오버레이)

---

### [UI-008] 수정 완료 후 성공 피드백 toast 없음

**심각도**: 중간
**발생 위치**: 이벤트 수정 제출 후

**현상**

- "수정 완료" 버튼 클릭 후 상세 페이지로 바로 리다이렉트
- 수정이 성공했다는 toast 알림이나 시각적 피드백이 없음

**문제점**

- 사용자가 수정이 성공했는지 실패했는지 즉각적으로 알기 어려움
- 이벤트 생성 시에도 동일한 문제 존재 가능

**개선 방향**

- Server Action 성공 시 toast 메시지 표시 ("이벤트가 수정되었습니다 ✓")
  - `useEffect`로 searchParams에서 success 플래그 감지하거나
  - Client Component에서 onSuccess 콜백 처리

## 낮음

### [UI-009] 이벤트 커버 이미지 없을 때 Placeholder 단조로움

**심각도**: 낮음
**발생 위치**: 이벤트 카드 및 상세 페이지

**현상**

- 커버 이미지가 없는 이벤트는 회색 gradient 배경에 캘린더 아이콘만 표시
- 카테고리별로 동일한 placeholder 사용

**문제점**

- 시각적 구분이 없어 여러 이벤트가 나란히 표시될 때 단조로워 보임

**개선 방향**

- 카테고리별 고유 색상 gradient placeholder 적용
  - 생일파티: 핑크/보라 gradient
  - 워크샵: 파란색 gradient
  - 스터디: 초록 gradient 등
- 또는 카테고리별 아이콘(케이크🎂, 📚, 🏃 등) 사용
- 최신 트렌드: 그라디언트 메시(gradient mesh) 배경으로 시각적 개성 부여
